# RateJudge - 審査員採点支援ツール

RateJudge は、ジャグリング大会などのパフォーマンス競技において、審査員がリアルタイムで採点を行い、即座に分析を行うための Web アプリケーションです。

**[デモ版 (GitHub Pages)](https://sugitoy.github.io/rate_judge/)**

## 主な特徴

- **リアルタイム採点**: スプレッドシート形式のインターフェースで、複数の選手と審査項目をスムーズに採点できます。
- **即時データ分析**: 入力したスコアに基づき、得点分布、項目別バランス、統計データなどを即座に可視化します。
- **複数大会・部門管理**: 複数の大会や部門（男子・女子、個人・チームなど）を切り替えて管理できます。
- **CSV インポート/エクスポート**: 大会設定や選手リスト、採点結果を CSV 形式で一括管理できます。
- **オフライン動作**: ブラウザのローカルストレージを使用するため、インターネット環境が不安定な会場でも動作します。

## クイックスタート（サンプルデータを使用した試用手順）

`testdata` ディレクトリにあるサンプルファイルを使用して、すぐに機能を試すことができます。

### 1. 大会のセットアップ
1. アプリ右上の **「設定」** タブをクリックします。
2. 右側のサイドメニューから **「新規大会追加」** をクリックします。
3. メインエリアの「CSVから大会設定を読み込む」にある **「ファイルを選択」** ボタンを押し、`testdata/大会設定_sample.csv` を選択します。
4. 続けて「CSVから選手リストを読み込む」から `testdata/選手リスト_sample.csv` を選択します。
5. 右側のサイドメニューの **「保存」** ボタンをクリックします。

### 2. 採点の実施
1. ヘッダー左側の **「採点」** タブをクリックします。
2. 表示されたテーブルに数値を入力します。数値は自動的に保存されます。
   - ※ `testdata/採点_sample.csv` をインポートして、既に入力済みの状態から試すことも可能です。

### 3. 結果の分析
1. ヘッダー左側の **「分析」** タブをクリックします。
2. グラフや統計テーブルが表示されます。
3. 右側のサイドメニューで選手を絞り込んだり、表示モード（得点/パーセント）を切り替えたりして、結果を詳細に確認できます。

## 開発者向けセットアップ

本プロジェクトをローカル環境で実行・ビルドする手順です。

### 実行環境
- Node.js (v18以上推奨)
- npm

### インストール
```bash
npm install
```

### 開発サーバーの起動
```bash
npm run dev
```

### ビルド
```bash
npm run build
```
