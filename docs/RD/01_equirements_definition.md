# 要件定義書 (Requirements Definition)

## 1. システム概要
本システムは、ジャグリング競技会における審査員が、選手の演技を迅速かつ正確に採点し、リアルタイムで結果を集計・分析するためのブラウザベースのツールです。

## 2. 背景と目的
従来の紙ベースや単純なスプレッドシートによる採点では、集計ミスやランキングの把握に時間がかかる課題がありました。本ツールは、直感的なUI（表形式入力、%とptの相互変換）と即時の視覚化機能、ローカル保存機能を提供することで、審査業務の効率化と正確性の向上を目的とします。

## 3. ユーザー特性
- **審査員**: 競技会中に手元で素早くスコアを入力する。
- **運営担当者**: 大会設定（選手リスト、審査項目）を事前に行い、最終結果を出力する。

## 4. 機能要件

### 4.1. 共通要件 (Common)
- **データ永続化**: ブラウザのLocalStorageを利用した自動保存（リロードしても消えない）。
- **データ初期化**: 全データのクリア機能（誤操作防止の警告ダイアログ付き）。
- **データ移行**: システムのアップデート時に古いデータ形式を検知し、バックアップの提供および新形式への移行を促す。
- **複数大会対応**: 複数の大会・部門データを保持し、ヘッダーから操作対象を随時切り替え可能。新規作成モード中であっても、大会セレクターで既存の大会が選択された場合は、その大会をアクティブにして編集モードに移行する。
- **ソート順の同期**: 選手の並び順（項目・昇順/降順）は全タブで同期され、採点テーブルおよび分析グラフの表示順に反映される。
- **状態保持**: タブ切り替え時やリロード時も、現在選択中の大会や入力途中の状態を維持する。
- **CSVインポート**: 大会基本情報、選手リスト、および外部で作成した採点データをCSVから取り込む機能。取り込み時にデータの不備やフォーマットの整合性チェックを行い、不正な場合はエラーメッセージを表示する。
- **CSVエクスポート**: 採点結果（合計・詳細・コメント含む）をCSV形式で保存・出力する機能。

### 4.2. 大会設定機能 (Configuration)
- **大会情報管理**: 大会名、部門名の設定。
- **審査項目設定**: 項目名と満点（最大スコア）の設定。
- **採点単位設定**: 採点の最小刻み（1, 0.5, 0.1単位）の設定。
- **選手管理**: 氏名、所属、使用道具の登録。
- **複数大会管理**: 複数の大会（または部門）を切り替えて管理。大会追加ボタン押下時は大会セレクターの選択をクリアし、未選択状態にする。
- **設定タブの遷移挙動**: ヘッダーの設定タブボタンを押した際、登録済み大会がない場合は新規作成モードで開き、登録済み大会がある場合は現在選択中の大会を編集モードで開く。

### 4.3. 採点機能 (Scoring)
- **スプレッドシート型入力**: 全選手と審査項目を一覧できるテーブル形式。
- **双方向入力モード**:
    - **得点入力 (pt)**: 絶対値での入力。
    - **割合入力 (%)**: 満点に対するパーセンテージでの入力（自動でptに変換）。
- **入力バリデーション**: 設定された採点単位に合致しない数値の警告。
- **リアルタイム集計**: 入力と同時に合計点および暫定順位を算出。順位は「合計順位」に加え、各「審査項目ごとの順位」も算出・表示する。
- **表示制御**: 採点テーブル上の順位（合計順位および項目別順位）の表示有無を随時切り替え可能。
- **表示モードの同期**: 「得点入力 (pt)」と「割合入力 (%)」の切り替え状態は、全タブ（採点・分析・詳細モーダル）で共通化され、同期される。
- **表示対象選手の絞り込み**: 採点テーブルに表示する選手を個別に選択可能。選択状態は分析画面と同期する。
- **ソート機能**: エントリーNo、合計点、または任意の審査項目に基づいて選手の並び順を変更可能。項目に応じて昇順（エントリーNo）または降順（合計点・審査項目）が自動的に適用される。
- **詳細入力/コメント**: 各選手への個別講評やメモの保存。
- **トップ3ハイライト**: 暫定1位〜3位の選手をアイコンと色で強調。

### 4.4. 分析機能 (Dashboard)
- **統計分析**: 各項目および合計値の平均、中央値、最大値、最小値、分散の算出。表示モード（pt/%）の切り替えに対応し、選択されたモードに応じて数値を表示。
- **スコア分布可視化**: 積み上げ棒グラフによる選手ごとの得点内訳表示。表示モード（pt/%）の切り替えに対応。
- **絞り込み表示**: グラフに表示する選手を任意に選択可能。選択状態は採点画面と同期する。
- **個別項目比較**: 審査項目ごとの得点分布を個別のグラフで表示。
- **選手詳細比較 (レーダーチャート)**: 選択した複数選手（最大5名等）の得点バランスを、満点に対する獲得割合で正規化したレーダーチャートとして比較表示。

## 5. 非機能要件
- **ユーザビリティ**:
    - 数値の視認性を高めるため、等幅フォント（tabular-nums）を使用。
    - 入力時のゼロ埋め自動除去（010 -> 10）。
    - 入力欄へのフォーカス時にテキストを全選択（スムーズな上書き操作を支援）。
    - 画面幅を最大限に活用したレイアウト（フルワイド対応）。
- **パフォーマンス**: 大人数（数十名規模）の選手リストでも遅延のないリアルタイム計算。
- **レスポンシブ**: PCでの操作を主眼としつつ、タブレット等のモバイルデバイスにも配慮したデザイン。
- **オフライン利用**: ブラウザ完結型であり、インターネット接続がない環境でも動作可能。
