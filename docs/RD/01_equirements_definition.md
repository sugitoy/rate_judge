# 要件定義書 (Requirements Definition)

## 1. システム概要
本システムは、ジャグリング競技会における審査員が、選手の演技を迅速かつ正確に採点し、リアルタイムで結果を集計・分析するためのブラウザベースのツールです。

## 2. 背景と目的
従来の紙ベースや単純なスプレッドシートによる採点では、集計ミスやランキングの把握に時間がかかる課題がありました。本ツールは、直感的なUI（表形式入力、%とptの相互変換）と即時の視覚化機能、ローカル保存機能を提供することで、審査業務の効率化と正確性の向上を目的とします。

## 3. ユーザー特性
- **審査員**: 競技会中に手元で素早くスコアを入力する。
- **運営担当者**: 大会設定（選手リスト、審査項目）を事前に行い、最終結果を出力する。

## 4. 機能要件

### 4.1. 共通要件 (Common)
- **データ永続化**: ブラウザのLocalStorageを利用した自動保存（リロードしても消えない）。
- **データ初期化**: 全データのクリア機能（誤操作防止の警告ダイアログ付き）。
- **データの継続性と移行**: 
    - ブラウザの LocalStorage を利用し、リフレッシュ後も状態を維持する。
    - **移行ロジック (Migration)**: システムのアップデート（スキーマ変更）時に古いデータ形式を検知した場合、自動的に現行形式へ変換を行う。
    - **安全策**: 移行実行前に、旧形式の全データを `rate_judge_migration_backup` キーで保存し、ユーザーが手動でバックアップ（JSONファイル）をダウンロードできる手段を提供する。
- **複数大会対応**: 複数の大会・部門データを保持し、ヘッダーから操作対象を随時切り替え可能。新規作成モード中であっても、大会セレクターで既存の大会が選択された場合は、その大会をアクティブにして編集モードに移行する。
- **ソート順の同期**: 選手の並び順（項目・昇順/降順）は全タブで同期され、採点テーブルおよび分析グラフの表示順に反映される。
- **状態保持**: タブ切り替え時やリロード時も、現在選択中の大会や入力途中の状態を維持する。
- **テーマ切り替え**: ライトモード（標準）とダークモード（ネオンテーマ）をユーザーが随時切り替え可能。設定状態はブラウザに保存される。
- **CSVインポート**: 大会基本情報、選手リスト、および外部で作成した採点データをCSVから取り込む機能。取り込み時にデータの不備やフォーマットの整合性チェックを行い、不正な場合はエラーメッセージを表示する。**採点データのインポート時には、取り込まれたスコアに基づき最も近いTierを自動的に算出し設定する。**
- **CSVエクスポート**: 採点結果（合計・詳細・コメント含む）をCSV形式で保存・出力する機能。

### 4.2. 大会設定機能 (Configuration)
- **大会情報管理**: 大会名、部門名の設定。
- **審査項目設定**: 項目名と満点（最大スコア）の設定。
- **採点単位設定**: 採点の最小刻み（1, 0.5, 0.1単位）の設定。
- **減点機能設定**: 減点の有無をチェックボックスで設定可能。採点単位設定の下に配置。有効にすると採点テーブルに減点入力欄が追加される。
- **選手管理**: 氏名、所属、使用道具の登録、および**失格設定**が可能。エントリーNoはリストの並び順に基づいて1から自動的に付与され、手動での編集は行わない。
- **複数大会管理**: 複数の大会（または部門）を切り替えて管理。**設定画面のサイドメニュー**から大会の選択および新規追加を行う。
- **失格機能**: 失格としてマークされた選手は、採点テーブルや分析画面（グラフ・統計）から完全に除外される。ただし、データとしては保持され、CSVエクスポート時には出力される。
- **設定タブの遷移挙動**: ヘッダーの設定タブボタンを押すと設定画面が開く。大会未選択または新規作成ボタン押下時は新規作成モード、大会選択時はその大会の編集モードとなる。

### 4.3. 採点機能 (Scoring)
- **スプレッドシート型入力**: 全選手と審査項目を一覧できるテーブル形式。
- **双方向入力モード**:
    - **得点入力 (pt)**: 絶対値での入力。
    - **割合入力 (%)**: 満点に対するパーセンテージでの入力（自動でptに変換）。
    - **Tier入力 (Tier)**: 規定の階層（S+〜G）を選択して入力。階層ごとに設定された割合に基づき、自動でptに変換される。
- **Tier入力の挙動**:
    - Tier選択時に割合・スコアを更新する。
    - **得点・割合入力時も、現在の数値に最も近い割合を持つTierを自動的に選択し同期する。**
    - 選択されているTierと現在の数値が一致しない場合（補正の影響等）は、警告を表示する（Tier入力モード時のみ）。
- **入力バリデーション**: 
    - **採点単位エラー**: 設定された採点単位（0.5単位等）に合致しない数値の警告。全モード（pt, %, Tier）で適用。
    - **Tier不一致エラー**: 選択されたTierと実数値が一致しない場合の警告。Tier入力モードで適用。
    - **視覚的フィードバック**: エラー発生時は入力ボックスの枠線を赤色で表示する。また、pt/%モードでは入力ボックスの背景・文字色も赤系で強調する。
- **小計・合計の区別**: 審査項目の得点合計を「小計」として表示する。減点が有効な場合は小計から減点を差し引いた値を「合計」として表示し、順位計算には「合計」を使用する。減点が無効な場合は小計と合計を同値として扱い、「合計」列のみ表示する。
- **減点入力**: 減点が有効な場合、採点テーブルの審査項目列の右隣に減点列を設ける。数値は絶対値（正の数）で入力する。採点単位に関わらず 0.1 単位での入力を許容する。
- **リアルタイム集計**: 入力と同時に合計点および暫定順位を算出。順位は「合計順位」に加え、各「審査項目ごとの順位」も算出・表示する。
- **表示制御**: 採点テーブルの表示を簡略化する「省略表示」モードを搭載。有効時は「各審査項目の順位」「小計」「減点」「合計順位」が非表示となり、主要な得点入力に特化したレイアウトとなる（合計点は常に表示される）。 |
- **表示モードの同期**: 「得点入力 (pt)」と「割合入力 (%)」の切り替え状態は、全タブ（採点・分析・詳細モーダル）で共通化され、同期される。
- **表示対象選手の絞り込み**: 採点テーブルに表示する選手を個別に選択可能。選択状態は分析画面と同期する。
- **ソート機能**: エントリーNo、合計点、小計、減点、または任意の審査項目に基づいて選手の並び順を変更可能。項目に応じて昇順（エントリーNo）または降順（合計点・審査項目等）が自動的に適用される。また、採点テーブルの各ヘッダーをクリックすることでもソートの切り替えが可能。
- **詳細入力/コメント**: 各選手への個別講評やメモの保存。**詳細モーダル内では、選手の現在の暫定順位（合計順位）を表示する。**
- **トップ3ハイライト**: 暫定1位〜3位の選手を専用の背景色と文字色で強調する（メダルアイコン等は使用しない）。

### 4.4. 分析機能 (Dashboard)
- **統計分析**: 各項目および合計値の平均、中央値、最大値、最小値、分散の算出。表示モード（pt/%）の切り替えに対応し、選択されたモードに応じて数値を表示。減点が有効な場合は、統計サマリーに「減点」行を追加し、他の審査項目と同様の統計値を算出・表示する。
- **減点の可視化**: 全体得点分布として2種類のグラフを表示する。①**小計グラフ**（審査項目の得点を積み上げた棒グラフ、減点は含まない）と②**合計グラフ**（減点が有効な場合のみ表示・減点を差し引いた最終的な合計得点を0から始まるシンプルな1本の棒グラフで表示する）。各項目別分布グラフにも「減点」の個別グラフを追加する。
- **スコア分布可視化**: 積み上げ棒グラフによる選手ごとの得点内訳表示。表示モード（pt/%）の切り替えに対応.
- **絞り込み表示**: グラフに表示する選手を任意に選択可能。選択状態は採点画面と同期する。
- **個別項目比較**: 審査項目ごとの得点分布を個別のグラフで表示。
- **選手リスト**:
  - 氏名、所属、使用道具を登録。
  - エントリー番号（出番順）の入れ替えが可能。
  - **失格設定**: 選手ごとに失格フラグを設定可能。
- **データ管理**:
  - CSVによる選手リストの一括インポート・エクスポート。
  - **失格選手の扱い**: 失格となった選手は採点テーブルや分析グラフ、統計（平均・順位等）から完全に除外される。ただし、生データのエクスポート時には情報保持のため出力される。
- **選手詳細比較 (レーダーチャート)**: 選択した複数選手（最大5名等）の得点バランスを、満点に対する獲得割合で正規化したレーダーチャートとして比較表示。

### 4.5. 入力制限とバリデーション
- **厳格なバリデーション**: システム全体の整合性を保つため、型や数値範囲のバリデーションを適用する。
- **入力規則の集約**: 各入力項目の具体的な規則（文字数、数値単位、補正ロジック等）は、基本設計書（BD）の「04. 画面詳細」にて定義し、実装に反映する。
- **CSV整合性**: CSVインポート時には、入力仕様書に基づいたフォーマットチェックを行い、不正なデータによるシステムの不安定化を防止する。

## 5. 非機能要件
- **ユーザビリティ**:
    - 数値の視認性を高めるため、等幅フォント（tabular-nums）を使用。
    - 入力時のゼロ埋め自動除去（010 -> 10）。
    - 入力欄へのフォーカス時にテキストを全選択（スムーズな上書き操作を支援）。
    - 画面幅を最大限に活用したレイアウト（フルワイド対応）。
    - **ハイコントラスト・デザイン**: ダークモード（ネオンテーマ）において、セクション境界（ヘッダー、サイドパネル等）を明るいラインで強調し、低照度環境下での誤認を防止する。
- **パフォーマンス**: 大人数（数十名規模）の選手リストでも遅延のないリアルタイム計算。
- **レスポンシブ**: PCでの操作を主眼としつつ、タブレット等のモバイルデバイスにも配慮したデザイン.
- **オフライン利用**: ブラウザ完結型であり、インターネット接続がない環境でも動作可能。
