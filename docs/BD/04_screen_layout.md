# 04. 画面詳細（構成・ロジック）

アプリケーションの具体的な構成と、各画面内の論理的なセクションごとの表示項目・ロジックを定義する。

## 1. 画面構成

アプリケーションは、トップレベルのナビゲーションとしてグローバルヘッダーを持ち、その下に3つの主要タブを表示する。

1.  **共通ヘッダー**: 常に上部に表示されるナビゲーション。
2.  **設定画面（Configuration）**: 大会の基本定義、審査項目、選手リストの管理および外部連携。
3.  **採点画面（Scoring）**: スプレッドシート形式の採点入力、詳細コメント、および結果の出力。
4.  **ダッシュボード（Dashboard）**: 統計計算および多角的なグラフ表示。

---

## 2. 画面詳細

### 2.1. 共通ヘッダー (Global Header)
全画面共通で表示され、アプリケーションの状態（操作対象の大会）を管理する。

#### A) ナビゲーション
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| アプリロゴ / タイトル | - | クリックで設定画面（デフォルト）へ遷移。 |
| タブナビゲーション | タブリンク | 「設定」「採点」「ダッシュボード」を切り替える。 |

#### B) 大会コンテキスト管理
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 大会セレクター | ドロップダウン | 操作対象の大会を切り替える。全タブの表示データが同期される。 |
| 大会追加ボタン | ボタン(+) | 設定画面を**新規作成モード**に切り替え、未入力のデータ枠を表示する。 |
| 全データ初期化 | ボタン | アプリ内の全大会データを消去する危険操作。確認ダイアログを挟む。 |

---

### 2.2. 設定画面 (Configuration Tab)
大会の骨組みを構築する画面。「新規作成モード」と「編集モード」でボタンの表示や保存時の挙動が変化する。

#### A) 大会基本情報・審査項目 (Basic Info & Criteria)
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| モード警告バナー | 表示のみ | **新規作成モード時のみ**画面上部に表示され、新しい大会を編集中であることを通知する。 |
| 大会名 | テキストボックス | 大会の正式名称。入力必須。 |
| 部門名 | テキストボックス | カテゴリ名（例：男子個人）。 |
| 採点単位設定 | ドロップダウン | 最小刻み（0.1, 0.5, 1等）を指定。採点時の補正やバリデーションに使用。 |
| 審査項目の追加/削除 | ボタン / リスト | 項目名と満点（pt）を定義。削除時は、その項目に関連する全選手のスコアが破棄される。 |
| 設定インポート/出力 | ボタン | 大会定義（項目名・満点等）のCSV入出力。インポート時は必須項目（大会名等）の存在チェックを行い、不正フォーマットの場合はエラーメッセージを表示して処理を中断する。 |

#### B) 選手管理 (Player Management)
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 選手リストの管理 | テーブル形式 | 氏名、所属、使用道具を登録。行の追加・削除が可能。 |
| 選手リストインポート/出力 | ボタン | 選手情報のCSV一括入出力。既存リストの置換または追加。空行のみの場合などは取り込みエラーとしてユーザーに通知する。 |

#### C) 画面操作 (Actions)
画面下部に固定配置されるボタン群。

| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 保存ボタン | ボタン | 画面上の変更を永続化する。<br>・**新規作成モード時**: 新しい大会として新規登録。<br>・**編集モード時**: 既存の大会データを上書き更新。 |
| キャンセルボタン | ボタン | **新規作成モード（かつ他にデータがある）時のみ表示**。作成を破棄して既存の大会表示に戻る。 |
| 削除ボタン | ボタン | **編集モード時のみ表示**。現在選択中の大会データを完全に削除する。 |

---

### 2.3. 採点画面 (Scoring Tab)
競技中のメイン入力画面。

#### A) 採点コントロール
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| スコア入力モード | トグル | 「数値(pt)」と「割合(%)」を切り替える。現在の表示単位のみを変更し、内部データには影響しない。 |
| スコアインポート/出力 | ボタン | 現在の採点結果（合計・詳細・コメント）をCSVで入出力。 |

#### B) 採点テーブル (Scoring Spreadsheet)
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 選手・スコアセル | テーブル形式 | **行**: 選手（左端固定）。**列**: 項目別、合計、順位。<br>・入力単位に合わない数値は警告表示。<br>・パーセンテージ入力時は `割合 * 満点` を指定単位で丸めて保存。 |
| 詳細・コメントボタン | アイコンボタン | クリックで詳細モーダルを開く。未入力ならグレー、入力済みなら青色に変化し視認性を確保。 |

#### C) 詳細モーダル (Detail Modal)
特定の選手にフォーカスして採点とコメント記入を行うためのポップアップ画面。

| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 選手名（タイトル） | - | 操作対象の選手名を大きく表示。 |
| モーダル内スコア入力 | リスト/テーブル | 全審査項目の入力フィールドを表示。テーブル形式と同様に、数値と割合の双方向入力が可能。 |
| コメント入力エリア | テキストエリア | 自由記述の講評やメモを入力。複数行に対応。 |
| 保存ボタン | ボタン | 変更内容（スコア・コメント）を確定し、モーダルを閉じる。採点テーブルおよびダッシュボードへ即座に反映される。 |

---

### 2.4. ダッシュボード画面 (Dashboard Tab)
結果の分析と可視化。

#### A) 統計分析 (Analytics)
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 統計サマリーテーブル | 表形式 | 平均、中央値、最大、最小、分散を表示。各項目および合計値に対して算出。 |

#### B) 比較制御 (Control)
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 表示選手フィルタ | チェックリスト | グラフに表示する選手を個別に切り替え。 |

#### C) 視覚化 (Charts)
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 全体得点分布 | 積み上げ棒グラフ | 選手ごとの得点内訳を視覚化。エントリーNo順で表示。 |
| 項目別比較 | レーダーチャート | 各項目を100%正規化したバランス比較。複数選手の特性を同時に表示。 |
| 個別配布グラフ | 小規模棒グラフ | 審査項目ごとの得点分布を個別に表示。 |
