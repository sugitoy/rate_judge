# 04. 画面詳細（構成・ロジック）

アプリケーションの具体的な構成と、各画面内の論理的なセクションごとの表示項目・ロジックを定義する。

## 1. 画面構成

アプリケーションは、トップレベルのナビゲーションとしてグローバルヘッダーを持ち、その下のコンテンツエリアは「メインコンテンツ」と「サイドコンテンツ」の2カラム形式で構成される。

1.  **共通ヘッダー**: 常に上部に表示されるナビゲーション。
2.  **メインコンテンツ**: 各機能の主要な表示・入力エリア（画面左側）。
3.  **サイドコンテンツ**: 各機能に応じた操作、フィルタ、設定変更等の補助エリア（画面右端）。

---

## 2. 画面詳細

### 2.1. 共通ヘッダー (Global Header)
全画面共通で表示され、アプリケーションの状態（操作対象の大会）を管理する。

**レスポンシブ挙動（ハンバーガーメニュー）**:
- **広い画面（通常時）**: タイトル・大会セレクター・タブナビ・クリアボタンをすべてヘッダー内に横並び表示する。
- **狭い画面（折り返しが発生するブレークポイント以下）**:
  - ヘッダー右端にハンバーガーアイコン（☰）ボタンを表示する。
  - タブナビと大会コンテキスト管理（大会セレクター・追加ボタン・全データ初期化）はハンバーガーメニューの中に格納し、通常時は非表示とする。
  - ハンバーガーボタンをクリックすると、メニューがヘッダー直下にドロップダウン表示される。
  - メニュー外をクリックするとメニューを閉じる。

#### A) ナビゲーション
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| アプリロゴ / タイトル | - | 常時表示。クリックで設定画面（デフォルト）へ遷移。 |
| タブナビゲーション | タブリンク | 広い画面では常時表示。狭い画面ではハンバーガーメニュー内に格納。 |
| ハンバーガーボタン(☰) | ボタン | 狭い画面のみ表示。クリックでメニューの展開/折り畳みを切り替える。 |

**設定タブボタンの挙動**:
- **登録されている大会がない場合**: 設定画面を**新規作成モード**で開く。
- **登録されている大会がある場合**: 設定画面を、現在選択中の大会の**編集モード**で開く。

#### B) 大会コンテキスト管理
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 大会セレクター | ドロップダウン | 操作対象の大会を切り替える。全タブの表示データが同期される。広い画面では常時表示、狭い画面ではメニュー内表示。 |
| 大会追加ボタン | ボタン(+) | 設定画面を**新規作成モード**に切り替え、未入力のデータ枠を表示する。同時に大会セレクターの選択をクリアし、未選択状態にする。 |
| 全データ初期化 | ボタン | アプリ内の全大会データを消去する危険操作。確認ダイアログを挟む。 |

---


### 2.2. 設定画面 (Configuration Tab)
大会の骨組みを構築する画面。「新規作成モード」と「編集モード」で表示内容が変化する。

#### A) 大会基本情報・審査項目
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| モード警告バナー | 表示のみ | **新規作成モード時のみ**画面上部に表示され、新しい大会を編集中であることを通知する。 |
| 大会名 | テキストボックス | 大会の正式名称。入力必須。 |
| 部門名 | テキストボックス | カテゴリ名（例：男子個人）。 |
| 採点単位設定 | ドロップダウン | 最小刻み（0.1, 0.5, 1等）を指定。採点時の補正やバリデーションに使用。 |
| 審査項目の追加/削除 | ボタン / リスト | 項目名と満点（pt）を定義。削除時は、その項目に関連する全選手のスコアが破棄される。 |
| 設定インポート/出力 | ボタン | 大会定義（項目名・満点等）のCSV入出力。 |

#### B) 選手管理
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 選手リストの管理 | テーブル形式 | 氏名、所属、使用道具を登録。行の追加・削除が可能。 |
| 選手リストインポート/出力 | ボタン | 選手情報のCSV一括入出力。既存リストの置換または追加。 |

#### C) サイドコンテンツ（操作・アクション）
画面右端に配置され、大会データの保存や削除を行う。

| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 保存ボタン | ボタン | 画面上の変更を永続化する。 |
| キャンセルボタン | ボタン | **新規作成モード時のみ表示**。作成を破棄して既存の大会表示に戻る。 |
| 削除ボタン | ボタン | **編集モード時のみ表示**。現在選択中の大会データを完全に削除する。 |

---

### 2.3. 採点画面 (Scoring Tab)
競技中のメイン入力画面。

#### A) メインコンテンツ
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 採点テーブル | テーブル形式 | **行**: 選手（左端固定）。**列**: 項目名、合計、順位。<br>・詳細・コメントボタンからモーダル展開。 |

#### B) サイドコンテンツ（コントロール）
画面右端に配置され、入力設定やデータの入出力を行う。

| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| スコア入力モード | トグル | 「数値(pt)」と「割合(%)」を切り替える。 |
| スコアインポート | ボタン | CSVから採点結果を読み込む。 |
| スコアエクスポート | ボタン | 現在の結果をCSVで出力。 |

#### C) 詳細モーダル (Detail Modal)
特定の選手にフォーカスして採点とコメント記入を行うためのポップアップ画面。

| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 選手名（タイトル） | - | 操作対象の選手名を大きく表示。 |
| モーダル内スコア入力 | リスト/テーブル | 全審査項目の入力フィールドを表示。テーブル形式と同様に、数値と割合の双方向入力が可能。 |
| コメント入力エリア | テキストエリア | 自由記述の講評やメモを入力。複数行に対応。 |
| 保存ボタン | ボタン | 変更内容（スコア・コメント）を確定し、モーダルを閉じる。採点テーブルおよび分析画面へ即座に反映される。 |

---

### 2.4. 分析画面 (Analysis Tab)
結果の分析と可視化。

#### A) 統計分析 (Analytics)
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 統計サマリーテーブル | 表形式 | 平均、中央値、最大、最小、分散を表示。 |

#### B) 全体得点分布
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 全体得点分布グラフ | 積み上げ棒グラフ | 選手ごとの得点内訳を視覚化。 |

#### C) 各項目別分布
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 項目別配布グラフ | 小規模棒グラフ | 審査項目ごとの得点分布を個別に表示。 |

#### D) レーダーチャート
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 項目別比較 | レーダーチャート | 各項目を100%正規化したバランス比較。 |

#### E) サイドコンテンツ（表示対象フィルタ）
画面右端に配置され、グラフに表示する対象を制御する。

| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 表示対象選手 | チェックリスト | グラフに表示する選手を個別に切り替え。 |
