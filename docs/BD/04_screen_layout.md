# 04. 画面詳細（構成・ロジック）

アプリケーションの具体的な構成と、各画面内の論理的なセクションごとの表示項目・ロジックを定義する。

## 1. 画面構成

アプリケーションは、トップレベルのナビゲーションとしてグローバルヘッダーを持ち、その下のコンテンツエリアは「メインコンテンツ」と「サイドコンテンツ」の2カラム形式で構成される。

1.  **共通ヘッダー**: 常に上部に表示されるナビゲーション。
    - **ライト**: `bg-white` と `border-slate-200` による清潔感のある構成。
    - **ダーク**: `bg-slate-900` と高コントラストな境界線（`border-slate-600`）および強力なシャドウを採用し、視認性を最大化。
2.  **メインコンテンツ**: 各機能の主要な表示・入力エリア。
    - **ライト**: `bg-slate-50` を採用。
    - **ダーク**: 極暗の `bg-slate-950` を採用し、コンテンツの発光エフェクトを際立たせる。
3.  **サイドコンテンツ**: 各機能に応じた操作、フィルタ、設定変更等の補助エリア。
    - **UI構成**: 画面右端に固定。ライトモードでは `border-l-slate-200`、ダークモードでは `border-l-slate-600` と強力なシャドウによりメイン領域と分離する。
    - **トグルボタン**: パネル左端に配置。ダークモード時には境界線を明るくし、最小化状態でも「光のライン」として存在を明示する。
    - **レスポンシブ挙動**: 画面幅が狭い（`lg` 未満）環境では、パネル展開時にトグルボタンが画面外に隠れないよう、幅を `calc(100% - 3rem)` 程度に制限する。

## 2. 画面詳細

### 2.1. 共通ヘッダー (Global Header)
全画面共通で表示され、アプリケーションの状態（操作対象の大会）を管理する。

**レスポンシブ挙動（ハンバーガーメニュー）**:
- **広い画面（通常時）**:
  - **左側**: タイトル、および「採点」「分析」タブを配置。
  - **右側**: 「設定」タブを配置。
- **狭い画面（折り返しが発生するブレークポイント以下）**:
  - ヘッダー右端にハンバーガーアイコン（☰）ボタンを表示する。
  - タブナビと大会コンテキスト管理（大会セレクター・追加ボタン・全データ初期化）はハンバーガーメニューの中に格納し、通常時は非表示とする。
  - ハンバーガーボタンをクリックすると、メニューがヘッダー直下にドロップダウン表示される。
  - メニュー外をクリックするとメニューを閉じる。

#### A) ナビゲーション
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| アプリロゴ / タイトル | - | 常時表示。ロゴの横に現在選択中の大会情報を「大会名 - 部門名」の形式で表示する。 |
| タブナビゲーション | タブリンク | 広い画面では常時表示。狭い画面ではハンバーガーメニュー内に格納。 |
| ハンバーガーボタン(☰) | ボタン | 狭い画面のみ表示。クリックでメニューの展開/折り畳みを切り替える。 |

**設定タブボタンの挙動**:
- **複数大会対応**: 複数の大会・部門データを保持し、**設定画面から**操作対象を随時切り替え可能。

#### B) 大会コンテキスト管理
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 大会セレクター | ドロップダウン | **設定画面のサイドメニューに配置**。操作対象の大会を切り替える。 |
| 大会追加ボタン | ボタン(+) | **設定画面のサイドメニューに配置**。新規作成モードに切り替える。 |
| 全データ初期化 | ボタン | **設定画面のサイドメニュー最下部に配置**。アプリ内の全大会データを消去する危険操作。確認ダイアログを挟む。 |

### 2.2. 設定画面 (Configuration Tab)
大会の骨組みを構築する画面。「新規作成モード」と「編集モード」で表示内容が変化する。

#### A) 大会基本情報・審査項目
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| モード警告バナー | 表示のみ | **新規作成モード時のみ**画面上部に表示され、新しい大会を編集中であることを通知する。 |
| **大会名** | テキストボックス | 大会の正式名称。1〜100文字。必須。 |
| **部門名** | テキストボックス | カテゴリ名（例：男子個人）。50文字以内。任意。 |
| **採点単位設定** | ドロップダウン | 最小刻み（0.1, 0.5, 1等）を指定。 |
| **減点設定** | チェックボックス | 採点単位設定の直下に配置。チェックを入れると採点テーブルに減点入力列が追加される。 |
| **審査項目の追加/削除** | ボタン / リスト | 項目名（1〜50文字）と満点（0.1〜1000pt）を定義。削除時は確認ダイアログを表示。 |

#### B) 選手リスト管理
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| **選手追加フォーム** | 入力・追加ボタン | 氏名、所属、使用道具を入力して追加。自動でエントリー番号を付与。 |
| **選手リスト** | テーブル形式 | 列: No、氏名、所属、使用道具、失格、削除。失格時はランキング計算から除外。 |
| **選手一括インポート** | CSVアップロード | CSVから一括登録。詳細は [06. 入力・計算ロジック](./06_input_specifications.md) 参照。 |

### 2.3. 採点画面 (Scoring Tab)
競技中のメイン入力画面。

#### A) メインコンテンツ（採点テーブル）

| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| **No** | 表示のみ | エントリー番号。ヘッダークリックでソート可能。 |
| **選手名 + 詳細** | 表示・ボタン | 選手名を表示。詳細ボタンから詳細モーダルを展開。 |
| **審査項目** | スコア入力 | 項目別得点と項目内順位を表示。ヘッダークリックでソート可能。 |
| **小計** | 表示のみ | 審査項目の合算値。有効時のみ表示。 |
| **減点** | 数値入力 | 絶対値で入力。有効時のみ表示。詳細は [06. 入力・計算ロジック](./06_input_specifications.md) 参照。 |
| **合計** | 表示のみ | 最終合計点（小計 - 減点）。ヘッダークリックでソート可能。 |
| **順位** | 表示のみ | 合計点に基づく暫定順位。1〜3位はハイライト表示。 |

**共通ロジック**:
- **ヘッダーソート**: ヘッダークリックで昇順/降順を切り替え。現在のソート状態をアイコンで表示。
- **除外処理**: 失格選手は表示・集計から除外。詳細は [02. データモデル](./02_data_model.md) 参照。
- **入力バリデーション**: 採点単位エラー、Tier不一致エラーを視覚的に警告（赤枠・メッセージ）。
- **表示制御（省略表示）**: 
    - 有効時は「項目別順位」「小計」「減点」列を非表示にする。
    - 合計順位列は表示を維持（幅 `w-24`）。各列の幅を圧縮して一覧性を高める。

#### B) サイドコンテンツ（コントロール）
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| スコア入力モード | トグル | 数値(pt)/割合(%)/Tierを切り替え。全画面で同期。 |
| 省略表示 | チェックボックス | 採点テーブルの表示密度を切り替え。 |
| 並び替え・表示対象 | 操作 | 選手の表示順および表示/非表示を個別に制御。分析画面と同期。 |
| インポート/出力 | ボタン | 採点データのCSV入出力。 |

#### C) 採点詳細モーダル
テーブルの選手名または詳細ボタンから開く、単一選手の詳細入力・閲覧画面。情報を高密度に整理した2カラムレイアウトを採用する。

- **ヘッダー**: 
    - 選手切り替え（前へ/次へ、プルダウン選択）を左側に配置。
    - 現在の「合計点」と「合計順位（バッジ表示）」を中央〜右側に強調表示。
    - 入力モード（pt/%/Tier）の切り替えトグルを右端に配置。
- **メインレイアウト (2カラム構成)**: 
    - **左カラム (スコア入力エリア)**: 審査項目および減点（有効時）を垂直リスト表示。各行は独立したカード形式。
    - **右カラム (講評エリア)**: 最大1000文字の自由記述テキストエリア。文字数カウント（例：`120 / 1000`）を表示する。
- **インタラクション**: 各入力内容はリアルタイムで自動保存される。
- **フッター**: 「閉じる」ボタンのみを配置。

### 2.4. 分析画面 (Analysis Tab)
#### A) 統計分析
平均、中央値、最大、最小、分散を表示。失格者を除外した母集団で計算。詳細は [06. 入力・計算ロジック](./06_input_specifications.md) 参照。

#### B) 分布グラフ
統計テーブル、合計得点分布、小計分布（積み上げ）、項目別分布、減点分布（有効時）、レーダーチャートの順で表示。レスポンシブな2カラムグリッドレイアウトを採用。

---

### 2.5. CSV仕様
外部データ取り込み時の整合性チェックルール。

- **失格判定**: `1`, `true` (または頭文字 `t`) は「失格」とみなす。それ以外は有効。
- **減点**: 負の数が入力された場合は絶対値として扱う。
- **自動補完**: 採点データのインポート時、入力されたスコアに基づき最も近いTierを自動的に算出して設定する。
- 詳細は [06. 入力・計算ロジック](./06_input_specifications.md) および [02. データモデル](./02_data_model.md) に集約。
