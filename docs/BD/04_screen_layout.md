# 04. 画面詳細（構成・ロジック）

アプリケーションの具体的な構成と、各画面内の論理的なセクションごとの表示項目・ロジックを定義する。

## 1. 画面構成

アプリケーションは、トップレベルのナビゲーションとしてグローバルヘッダーを持ち、その下のコンテンツエリアは「メインコンテンツ」と「サイドコンテンツ」の2カラム形式で構成される。

1.  **共通ヘッダー**: 常に上部に表示されるナビゲーション。
2.  **メインコンテンツ**: 各機能の主要な表示・入力エリア。最大幅を制限し、サイドパネルを除いた領域の中央に配置される。
3.  **サイドコンテンツ**: 各機能に応じた操作、フィルタ、設定変更等の補助エリア。**画面の右端（Viewport右端）に固定**され、縦幅全体を使用する。
    - **UI構成**: 画面占有率と操作性を優先し、サイドパネル最上部の見出し（アクション、採点コントロール等）は省略する。
    - **トグルボタン**: パネルの左端に突き出した「タブ」形式で配置され、最小化状態でも視認性と操作性を確保する。
    - **レスポンシブ挙動**: 画面幅が狭い（`lg` ブレークポイント未満）環境では、パネル展開時にトグルボタンが画面外に隠れないよう、幅を `calc(100% - 3rem)` 程度に制限する。

## 2. 画面詳細

### 2.1. 共通ヘッダー (Global Header)
全画面共通で表示され、アプリケーションの状態（操作対象の大会）を管理する。

**レスポンシブ挙動（ハンバーガーメニュー）**:
- **広い画面（通常時）**:
  - **左側**: タイトル、および「採点」「分析」タブを配置。
  - **右側**: 「設定」タブを配置。
- **狭い画面（折り返しが発生するブレークポイント以下）**:
  - ヘッダー右端にハンバーガーアイコン（☰）ボタンを表示する。
  - タブナビと大会コンテキスト管理（大会セレクター・追加ボタン・全データ初期化）はハンバーガーメニューの中に格納し、通常時は非表示とする。
  - ハンバーガーボタンをクリックすると、メニューがヘッダー直下にドロップダウン表示される。
  - メニュー外をクリックするとメニューを閉じる。

#### A) ナビゲーション
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| アプリロゴ / タイトル | - | 常時表示。ロゴの横に現在選択中の大会情報を「大会名 - 部門名」の形式で表示する。クリックで設定画面へ遷移。 |
| タブナビゲーション | タブリンク | 広い画面では常時表示。狭い画面ではハンバーガーメニュー内に格納。 |
| ハンバーガーボタン(☰) | ボタン | 狭い画面のみ表示。クリックでメニューの展開/折り畳みを切り替える。 |

**設定タブボタンの挙動**:
- **複数大会対応**: 複数の大会・部門データを保持し、**設定画面から**操作対象を随時切り替え可能。新規作成モード中であっても、設定画面のセレクターで既存の大会が選択された場合は、その大会をアクティブにして編集モードに移行する。

#### B) 大会コンテキスト管理
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 大会セレクター | ドロップダウン | **設定画面のサイドメニューに配置**。操作対象の大会を切り替える。 |
| 大会追加ボタン | ボタン(+) | **設定画面のサイドメニューに配置**。新規作成モードに切り替える。 |
| 全データ初期化 | ボタン | **設定画面のサイドメニュー最下部に配置**。アプリ内の全大会データを消去する危険操作。確認ダイアログを挟む。 |

### 2.2. 設定画面 (Configuration Tab)
大会の骨組みを構築する画面。「新規作成モード」と「編集モード」で表示内容が変化する。

#### A) 大会基本情報・審査項目
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| モード警告バナー | 表示のみ | **新規作成モード時のみ**画面上部に表示され、新しい大会を編集中であることを通知する。 |
| 大会名 | テキストボックス | 大会の正式名称。入力必須。 |
| 部門名 | テキストボックス | カテゴリ名（例：男子個人）。 |
| 採点単位設定 | ドロップダウン | 最小刻み（0.1, 0.5, 1等）を指定。採点時の補正やバリデーションに使用。 |
| 減点設定 | チェックボックス | 採点単位設定の直下に配置。チェックを入れると採点テーブルに減点入力列が追加される。デフォルトは無効（未チェック）。 |
| 審査項目の追加/削除 | ボタン / リスト | 項目名と満点（pt）を定義。削除時は、その項目に関連する全選手のスコアが破棄される。削除アイコンはデフォルトでグレー表示、ホバー時に赤（danger）表示にする。 |
| 設定インポート/出力 | ボタン | 大会定義（項目名・満点等）のCSV入出力。 |

#### B) 選手リスト管理
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 選手追加フォーム | 入力・追加ボタン | 氏名（必須）、所属、使用道具を入力して追加。追加時、自動でエントリー番号（連番）を付与。 |
| 選手リスト | テーブル形式 | **列**: No、氏名、所属、使用道具、失格、削除。<br>・**ドラッグ&ドロップ**: 並び替えによりエントリー順（No）を更新。<br>・**失格設定**: チェックボックス。削除ボタンの左側に配置。チェックを入れると採点対象外（ランキング除外）となる。 |
| 選手一括インポート | CSVアップロード | 氏名、所属、道具をCSVから一括登録。既存リストは上書き。 |

#### C) サイドコンテンツ（操作・アクション）
画面右端に配置され、大会データの保存や削除を行う。

| 大会セレクター | ドロップダウン | 編集・閲覧対象の大会を切り替える。 |
| 新規大会追加 | ボタン | 新規作成モードに切り替え、未入力のデータ枠を表示する。 |
| 保存ボタン | ボタン | 画面上の変更を永続化する。 |
| キャンセルボタン | ボタン | **新規作成モード時のみ表示**。作成を破棄して既存の大会表示に戻る。削除ボタンと同じ赤（danger）系スタイルで表示する。 |
| 削除ボタン | ボタン | **編集モード時のみ表示**。現在選択中の大会データを完全に削除する。 |
| 全データ初期化 | ボタン | **サイドメニュー最下部に配置**。アプリ内の全データを消去する。他の項目と距離を置いて配置する。 |

### 2.3. 採点画面 (Scoring Tab)
競技中のメイン入力画面。

#### A) メインコンテンツ
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
#### A) メインコンテンツ（採点テーブル）

| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| **No** | 表示のみ | エントリー番号。左端に固定。 |
| **選手名 + 詳細** | 表示・ボタン | 選手名を表示。右端に詳細ボタン（拡大アイコン）を配置し、詳細モーダルを展開する。左端に固定。 |
| **審査項目** | スコア入力 | 設定された全項目。入力単位に基づき数値を入力。値の横に、その項目内での順位を括弧書きで表示（例：`50.0 (3位)`）。 |
| **小計** | 表示のみ | 審査項目の合算値。減点設定が有効な場合のみ表示される。 |
| **減点** | 数値入力 | 減点値。絶対値（正の数）で入力。列幅を狭く（w-20）配置。減点設定が有効な場合のみ表示される。 |
| **合計** | 表示のみ | 最終合計点（`小計 - 減点`）。プライマリーカラーで強調表示。 |
| **順位** | 表示のみ | 合計点に基づく暫定順位。1〜3位は専用バッジ（金・銀・銅）を表示。 |

**共通ロジック**:
- **除外処理**: **「失格」マークがついた選手はテーブルに表示されず、順位計算の母集団（ランキング計算）や統計からも完全に除外される。**
- **リアルタイム算出**: 値の変更ごとに項目別順位、小計、合計点、合計順位を即座に再計算して反映する。
- **入力バリデーション（警告表示）**:
    - **対象**: 「採点単位エラー（全モード）」および「Tier不一致エラー（Tierモードのみ）」。
    - **枠線 (Border)**: 入力モードによらず、エラー発生時は入力ボックスの枠線を赤色（danger）にする。
    - **背景・文字色 (Background/Text)**: pt入力および%入力モードの場合のみ、エラー発生時に入力ボックスの背景色と文字色を赤系にする。Tierモードではセレクトボックス自体の背景・文字色は変更しない。
    - **メッセージ**: 入力ボックスの下部にエラー内容（「採点単位エラー」等）を赤文字で表示する。
- **表示制御**: サイドパネルの操作により「順位」の表示/非表示、および「表示対象選手」の個別絞り込みが可能。 |

#### B) サイドコンテンツ（コントロール）
画面右端に配置され、入力設定やデータの入出力を行う。

| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| スコア入力モード | トグル | 「数値(pt)」「割合(%)」「Tier」を切り替える。全タブ（分析、詳細モーダル含む）で同期される。<br>・割合/Tier入力時は入力単位に従いリアルタイムでスコアを補正し、前後の近い値（等距離なら低い方）に丸める。<br>・**Tier定義**: S+(95%), S(90%), A+(85%), A(80%), B(75%), C(70%), D(65%), E(60%), F(55%), G(50%)。
| 順位の表示 | チェックボックス | 合計順位列および項目別順位の表示/非表示を切り替える。 |
| 選手の並び順 | プルダウン | 並び替えの基準（エントリーNo、合計点、各審査項目）を選択。項目に応じて昇順/降順が自動設定される（Noは昇順、点数は降順）。 |
| 表示対象選手 | チェックリスト | 採点テーブルに表示する選手を個別に切り替え。分析画面のフィルタと同期する。 |
| スコアインポート | ボタン | **サイドメニュー最下部に配置**。CSVから採点結果を読み込む。 |
| スコアエクスポート | ボタン | **サイドメニュー最下部に配置**。現在の結果をCSVで出力。 |

#### C) 詳細モーダル (Detail Modal)
特定の選手にフォーカスして採点とコメント記入を行うためのポップアップ画面。

| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 選手ナビゲーション | ボタン（左右） | タイトル横に配置。前後のエントリーNoの選手へ表示を切り替える。切り替え時は現在の入力内容を自動保存する。 |
| 選手名・番号・順位・合計 | 表示のみ | 「No.XX 選手名」に加え、現在の「合計点」と「暫定順位」をタイトルエリアに強調表示する。 |
| スコアサマリー | 表示のみ | スコア入力リストの最後、またはタイトルエリア付近に「減点（有効時のみ）」「合計」のサマリーを表示し、現在の計算結果を即座に確認可能にする。 |
| レイアウト構成 | 2カラム | 画面左側に「スコア入力エリア」、右側に「コメント入力エリア」を配置し、情報の整理と一覧性を高める。見出し（スコア入力、コメント等）は省略し、情報の密度を高める。 |
| モーダル内スコア入力 | 垂直リスト | 全審査項目を縦一列に配置。項目名と入力UIを1行（横並び）に配置する。<br>・**Tierモード**: 利用可能なTierを横一列に並べ、ボタン形式で選択可能にする。<br>・**数値(pt)/割合(%)モード**: pt入力欄と%入力欄を横に並べて配置する。サイドパネルで指定されている入力方法（ptまたは%）のみを活性化し、もう一方は読み取り専用（または非活性）とする。<br>・減点が有効な場合は、リストの最後に「**減点**」行を設ける。 |
| コメント入力エリア | テキストエリア | 自由記述の講評やメモを入力。見出しを省略し、垂直方向のスペースを最大限活用する。以前の設計より高さを抑え（例：12〜16行程度）、他の要素とのバランスを調整する。入力内容はリアルタイムで自動保存される。 |
| 完了ボタン | ボタン | モーダルを閉じる。モーダル外側のクリックでも閉じることが可能。 |

### 2.4. 分析画面 (Analysis Tab)
結果の分析と可視化。

#### A) 統計分析 (Analytics)
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 統計サマリーテーブル | 表形式 | 平均、中央値、最大、最小、分散を表示。表示モード（pt/%）に応じて単位と値を切り替える。減点が有効な場合は、審査項目行の後に「**減点**」行を追加し、他項目と同様の統計値を算出・表示する。%表示時の減点は全審査項目の満点合計を分母として割合を算出する。 |

#### B) 全体得点分布
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 全体得点分布（小計）グラフ | 積み上げ棒グラフ | 選手ごとの**審査項目得点のみ**を積み上げ表示（減点は含まない）。表示モード（pt/%）に応じて単位と値を切り替える（%時は全審査項目の満点合計に対する割合）。 |
| 全体得点分布（合計）グラフ | 棒グラフ | 最終的な合計得点をシンプルな棒グラフで表示（0固定開始）。減点が有効な場合のみ、小計グラフと並べて表示する。 |

#### C) 各項目別分布
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 項目別配布グラフ | 小規模棒グラフ | 審査項目ごとの得点分布を個別に表示。表示モード（pt/%）に応じて単位と値を切り替える（%時は各項目の満点に対する割合）。減点が有効な場合は「**減点**」の個別グラフも追加表示する。 |

#### D) レーダーチャート
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 項目別比較 | レーダーチャート | 各項目を100%正規化したバランス比較。 |

#### E) サイドコンテンツ（表示対象フィルタ）
画面右端に配置され、グラフに表示する対象を制御する。

| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 表示モード | トグル | 「数値(pt)」と「割合(%)」を切り替える。採点画面と同期する。 |
| 選手の並び順 | プルダウン | 並び替えの基準を選択。項目に応じた順序が自動設定され、各グラフの表示順に反映される。 |
| 表示対象選手 | チェックリスト | グラフに表示する選手を個別に切り替え。採点画面と同期する。 |
