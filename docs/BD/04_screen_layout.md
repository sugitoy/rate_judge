# 04. 画面詳細（構成・ロジック）

アプリケーションの具体的な構成と、各画面内の論理的なセクションごとの表示項目・ロジックを定義する。

## 1. 画面構成

アプリケーションは、トップレベルのナビゲーションとしてグローバルヘッダーを持ち、その下のコンテンツエリアは「メインコンテンツ」と「サイドコンテンツ」の2カラム形式で構成される。

1.  **共通ヘッダー**: 常に上部に表示されるナビゲーション。
    - **ライト**: `bg-white` と `border-slate-200` による清潔感のある構成。
    - **ダーク**: `bg-slate-900` と高コントラストな境界線（`border-slate-600`）および強力なシャドウを採用し、視認性を最大化。
2.  **メインコンテンツ**: 各機能の主要な表示・入力エリア。
    - **ライト**: `bg-slate-50` を採用。
    - **ダーク**: 極暗の `bg-slate-950` を採用し、コンテンツの発光エフェクトを際立たせる。
3.  **サイドコンテンツ**: 各機能に応じた操作、フィルタ、設定変更等の補助エリア。
    - **UI構成**: 画面右端に固定。ライトモードでは `border-l-slate-200`、ダークモードでは `border-l-slate-600` と強力なシャドウによりメイン領域と分離する。
    - **トグルボタン**: パネル左端に配置。ダークモード時には境界線を明るくし、最小化状態でも「光のライン」として存在を明示する。
    - **レスポンシブ挙動**: 画面幅が狭い（`lg` 未満）環境では、パネル展開時にトグルボタンが画面外に隠れないよう、幅を `calc(100% - 3rem)` 程度に制限する。

## 2. 画面詳細

### 2.1. 共通ヘッダー (Global Header)
全画面共通で表示され、アプリケーションの状態（操作対象の大会）を管理する。

**レスポンシブ挙動（ハンバーガーメニュー）**:
- **広い画面（通常時）**:
  - **左側**: タイトル、および「採点」「分析」タブを配置。
  - **右側**: 「設定」タブを配置。
- **狭い画面（折り返しが発生するブレークポイント以下）**:
  - ヘッダー右端にハンバーガーアイコン（☰）ボタンを表示する。
  - タブナビと大会コンテキスト管理（大会セレクター・追加ボタン・全データ初期化）はハンバーガーメニューの中に格納し、通常時は非表示とする。
  - ハンバーガーボタンをクリックすると、メニューがヘッダー直下にドロップダウン表示される。
  - メニュー外をクリックするとメニューを閉じる。

#### A) ナビゲーション
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| アプリロゴ / タイトル | - | 常時表示。ロゴの横に現在選択中の大会情報を「大会名 - 部門名」の形式で表示する。クリックで設定画面へ遷移。 |
| タブナビゲーション | タブリンク | 広い画面では常時表示。狭い画面ではハンバーガーメニュー内に格納。 |
| ハンバーガーボタン(☰) | ボタン | 狭い画面のみ表示。クリックでメニューの展開/折り畳みを切り替える。 |

**設定タブボタンの挙動**:
- **複数大会対応**: 複数の大会・部門データを保持し、**設定画面から**操作対象を随時切り替え可能。新規作成モード中であっても、設定画面のセレクターで既存の大会が選択された場合は、その大会をアクティブにして編集モードに移行する。

#### B) 大会コンテキスト管理
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| 大会セレクター | ドロップダウン | **設定画面のサイドメニューに配置**。操作対象の大会を切り替える。 |
| 大会追加ボタン | ボタン(+) | **設定画面のサイドメニューに配置**。新規作成モードに切り替える。 |
| 全データ初期化 | ボタン | **設定画面のサイドメニュー最下部に配置**。アプリ内の全大会データを消去する危険操作。確認ダイアログを挟む。 |

### 2.3. 採点画面 (Scoring Tab)
競技中のメイン入力画面。

#### A) メインコンテンツ
| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
#### A) メインコンテンツ（採点テーブル）

| 表示項目 | 入力方式 | 詳細・ロジック |
| :--- | :--- | :--- |
| **No** | 表示のみ | エントリー番号。左端に固定。ヘッダーをクリックすることでエントリーNo順（昇順/降順）にソート可能。 |
| **選手名 + 詳細** | 表示・ボタン | 選手名を表示。右端に詳細ボタン（拡大アイコン）を配置し、詳細モーダルを展開する。左端に固定。 |
| **審査項目** | スコア入力 | 設定された全項目。0以上、満点以下。入力単位に基づき数値を入力。単位外は警告表示。値の横に、その項目内での順位を括弧書きで表示（例：`50.0 (3位)`）。ヘッダーをクリックすることで項目得点順（降順/昇順）にソート可能。 |
| **小計** | 表示のみ | 審査項目の合算値。減点設定が有効な場合のみ表示される。ヘッダーをクリックすることで小計得点順（降順/昇順）にソート可能。 |
| **減点** | 数値入力 | 減点値。0以上。絶対値（正の数）で入力。常に `0.1` 単位での入力を許容する。列幅を狭く（w-20）配置。減点設定が有効な場合のみ表示される。ヘッダーをクリックすることで減点値順（降順/昇順）にソート可能。 |
| **合計** | 表示のみ | 最終合計点（`小計 - 減点`）。プライマリーカラーで強調表示。ヘッダーをクリックすることで合計得点順（降順/昇順）にソート可能。 |
| **順位** | 表示のみ | 合計点に基づく暫定順位。1〜3位は専用バッジ（金・銀・銅）を表示。 |

**共通ロジック**:
- **ヘッダーソート**: 採点テーブルの「No」「審査項目」「小計」「減点」「合計」のヘッダーをクリックすることで、その項目をキーとした並び替えを行う。
    - **トグル挙動**: 現在のソートキーと異なるヘッダーをクリックした際は、各項目のデフォルト順序（Noは昇順、点数・減点系は降順）を適用する。同じヘッダーを連続してクリックした場合は、昇順・降順を反転させる。
    - **同期**: サイドパネルの「並び順」設定および他タブ（分析画面）の並び順と即座に同期される。
    - **視覚的フィードバック**: ソート対象のヘッダーにはソート方向（昇順/降順）を示すアイコンを表示し、クリック可能（ホバー時の背景色変化等）であることを明示する。
- **除外処理**: **「失格」マークがついた選手はテーブルに表示されず、順位計算の母集団（ランキング計算）や統計からも完全に除外される。**
- **リアルタイム算出**: 値の変更ごとに項目別順位、小計、合計点、合計順位を即座に再計算して反映する。
- **入力バリデーション（警告表示）**:
    - **対象**: 「採点単位エラー（全モード）」および「Tier不一致エラー（Tierモードのみ）」。
    - **枠線 (Border)**: 入力モードによらず、エラー発生時は入力ボックスの枠線を赤色（danger）にする。
    - **背景・文字色 (Background/Text)**: pt入力および%入力モードの場合のみ、エラー発生時に入力ボックスの背景色と文字色を赤系にする。Tierモードではセレクトボックス自体の背景・文字色は変更しない。
    - **メッセージ**: 入力ボックスの下部にエラー内容（「採点単位エラー」等）を赤文字で表示する。
- **表示制御**: サイドパネルの操作により「省略表示」への切り替えが可能。省略表示有効時は、各審査項目内の順位、小計、および減点列が非表示となる。また、各列の幅を圧縮して余白を最小化する。なお、合計順位列は常に表示され、幅のみが調整される。 |

### 2.5. CSVインポート・エクスポート仕様

外部データ取り込み時の整合性チェックルール。

#### A) 共通ルール
- **文字コード**: UTF-8 (BOM付き推奨)。
- **数値形式**: 全角数字は半角に変換、またはエラーとして扱う。カンマ等の区切り文字は排除。
- **エスケープ処理**: エクスポート時、データ内にカンマ(`,`)、改行(`\n`)、ダブルクォート(`"`)が含まれる場合は、値をダブルクォートで囲み、内部のダブルクォートは二重化(`""`)してエスケープする。`null` や `undefined` は空文字として出力する。
- **ファイル名規則**: 
    - 大会名などをファイル名に使用する際、OSで禁止されている文字（`\ / : * ? " < > |`）はアンダースコア（`_`）に置換してサニタイズする。
    - 各ファイル種別ごとに以下の接頭辞を付与する。
        - 大会設定: `01_大会設定_[大会名].csv`
        - 選手リスト: `02_選手リスト_[大会名].csv`
        - 採点結果: `03_採点_[大会名].csv`

#### B) バリデーションエラー・警告時の挙動
- **採点データインポート時の警告**: 既存のスコアデータが上書きされるため、実行前にユーザーに確認ダイアログを表示する。
- **インポートエラー時の通知**: CSVのパース失敗やバリデーション違反が発生した際、エラー原因（詳細メッセージ等）を含むアラートを表示し、処理を中断する。
- **必須項目の欠損**: インポートを中断し、エラー行数と項目名を表示。
- **型不一致**: 数値項目に非数値が含まれる場合、インポートを中断。
- **論理矛盾**: 
    - 満点設定がない項目へのスコア入力：エラー。
    - 存在しない選手ID/エントリーNoへの紐付け：エラー。

#### C) 特殊な値の解釈
- **ヘッダー行の自動検知**: 選手インポート時、1行目に「氏名」「所属」「道具」などのキーワードが含まれる場合は、ヘッダーとして解釈しデータとしての取り込みをスキップする。
- **失格列**: `1`, `true`, `yes` は「失格」とみなす。それ以外（空欄含む）は `0`（有効）。
- **減点列**: 負の数が入力された場合は絶対値として扱うか、またはエラーとする。無効な値は `0`。
