# 06. 入力・計算ロジック詳細仕様

本ドキュメントでは、システムの再現性を確保するため、数値補正、ティア選択、データ移行、CSV解析等の詳細なロジックを定義する。

## 1. 数値補正・丸めロジック

### 1.1. 単位補正 (Auto Correct)
割合（%）入力や、異なる単位での再計算時に、設定された「採点単位」への補正を行う。

- **基本ルール**: 
    - 入力値を採点単位で割り、最も近い整数倍の値を算出する。
    - **タイブレーク**: 上下の値への乖離が等しい（中間値）場合は、**低い方（床方向）**に寄せる。
- **浮動小数点誤差の取り扱い**:
    - 比較演算において、`1e-10` (0.0000000001) の閾値を設け、微小な誤差による意図しない切り上げ・切り捨てを防止する。
    - 例: `diff <= 0.5 + 1e-10` の条件で判定し、微小な超過を許容して低い方に丸める。

### 1.2. 表示フォーマット
- UI上の表示（スコア、統計値、ツールチップ）は、一貫して `toFixed(1)` を用いて小数点以下第1位まで表示する。
- 内部データは高精度な数値を保持し、累積誤差を最小限に抑える。

## 2. ティア選択ロジック

### 2.1. 最も近いティアの選定 (getClosestTier)
数値入力（pt/%）時に、現在の割合に最も近い定義済みティアを自動選択する。

- **選定ルール**: 
    - 各ティアの定義割合と、現在の入力割合の絶対差を比較し、最小のものを選択する。
    - **タイブレーク**: 乖離が等しい中間値の場合（例: 92.5%）、定義リストの**前方にあるティア（より高いランク）**を優先する。
- **ティア定義一覧**:
    - S+: 95%, S: 90%, A+: 85%, A: 80%, B: 75%, C: 70%, D: 65%, E: 60%, F: 55%, G: 50%

## 3. CSV解析・バリデーション仕様

### 3.1. 文字列の解釈
- **失格判定**: 
    - 以下の値（大文字小文字不問）を `isDisqualified: true` と判定する。
    - `1`, `true`, `t`
    - 上記以外（空欄、`0`, `false`, `f` 等）はすべて `false` とみなす。
- **数値解析**: 
    - 数値項目にカンマや単位が含まれる場合は排除して解析する。
    - 減点値に負の数が入力された場合は、絶対値として取り扱う。

### 3.2. 自動補完
- **採点データインポート**: 
    - インポートされたスコアに基づき、2.1 のロジックを用いて最も近いティアを全審査項目に対して自動的に設定する。

## 4. データ整合性・移行仕様

### 4.1. バージョン管理と移行 (Migration)
LocalStorage 内のデータを安全に移行する。

- **バージョン 3 への移行**:
    - 移行前データのフルバックアップを `rate_judge_migration_backup` キーで保存する。
    - `hasDeduction` フィールドが存在しない古い大会データに対し、デフォルト値として `false` を付加する。

## 5. 統計計算仕様

### 5.1. 母集団の定義
- **失格者の除外**: 
    - すべての集計処理（平均、中央値、分散、最大、最小、ランキング、グラフ表示）において、失格フラグが有効な選手を計算の直前で除外する。
